## 版本策略

### 语义化版本（SemVer）

```
版本号格式：v主版本.次版本.修订版本.构建版本

示例：v0.1.1.1
├─ v0：主版本号（Major）- 0表示开发中，1+表示稳定版
├─ 1：次版本号（Minor）- 功能添加，向后兼容
├─ 1：修订版本号（Patch）- Bug修复，向后兼容
└─ 1：构建版本号（Build）- 构建级别修改
```

### 版本类型

| 类型 | 说明 | 示例 | 破坏性变更 |
|------|------|------|------------|
| **Major（主版本）** | 重大架构变更，不兼容的API修改 | v1.0.0 | ⚠️ 可能有 |
| **Minor（次版本）** | 新功能添加，向后兼容 | v0.2.0 | ✅ 无 |
| **Patch（修订版）** | Bug修复，向后兼容 | v0.1.2 | ✅ 无 |
| **Build（构建版）** | 构建级别修改 | v0.1.1.1 | ✅ 无 |
