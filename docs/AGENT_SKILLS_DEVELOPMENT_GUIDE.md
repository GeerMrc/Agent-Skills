#### 架构设计原则

##### 原则1: 渐进式披露三层架构（PDA Pattern）

```
┌─────────────────────────────────────────────────────────────┐
│  PDA 三层架构                                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第一层：YAML frontmatter（元数据层）                        │
│  └─ 技能元数据、依赖声明、配置信息                           │
│                                                             │
│  第二层：SKILL.md（入口点层，≤200行）                        │
│  ├─ 技能核心描述                                            │
│  ├─ 快速引用列表                                            │
│  ├─ 关键功能概述                                            │
│  └─ 按需扩展链接                                            │
│                                                             │
│  第三层：references/ + scripts/ + templates/（详细内容层）   │
│  ├─ references/   - 详细文档（按需加载）                    │
│  ├─ scripts/      - 工具脚本（按需调用）                    │
│  └─ templates/    - 项目模板（按需生成）                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

##### 原则2: 严格200行规则

| 规则 | 要求 | 验证方式 |
|------|------|----------|
| **SKILL.md行数** | ≤200行 | `wc -l SKILL.md` |
| **内容完整性** | 100% | 人工验证 |
| **可读性** | A级以上 | 人工评估 |

##### 原则3: 一级引用深度

| 规则 | 要求 | 示例 |
|------|------|------|
| **最大深度** | SKILL.md → references/ → 文档 | 2层 |
| **禁止嵌套** | references/ 内部文档禁止互引 | ❌ 错误 |
| **独立文档** | 每个主题独立文档 | ✅ 正确 |

#### 推荐项目结构

```yaml
project-root/
├── SKILL.md                           # 技能入口（≤200行）
├── skill-name/                        # 主技能目录
│   ├── references/                    # 详细文档目录
│   │   ├── topic-1/                   # 按功能分类
│   │   ├── topic-2/
│   │   └── topic-3/
│   ├── scripts/                       # 工具脚本
│   │   ├── check-*.py                # 验证工具
│   │   ├── generate-*.py             # 生成工具
│   │   └── shared-modules/           # 共享模块
│   ├── templates/                     # 项目模板
│   │   ├── template-1/
│   │   └── template-2/
│   └── tests/                         # 测试文件
│       ├── unit/
│       ├── integration/
│       └── e2e/
├── docs/                              # 项目文档
│   ├── DEVELOPMENT_WORKFLOW.md
│   └── COMPREHENSIVE_AUDIT_PLAN.md
└── .claude/                           # Claude配置
```

#### 实施要点

1. ✅ **PDA架构**：采用渐进式披露三层架构
2. ✅ **200行规则**：SKILL.md 严格控制在200行以内
3. ✅ **一级引用**：仅允许一级引用深度
4. ✅ **功能导向**：按功能模块组织目录
5. ✅ **命名规范**：使用 kebab-case

#### 质量标准

- ✅ SKILL.md行数：≤200行
- ✅ YAML frontmatter：完整
- ✅ PDA架构：完整实现
- ✅ 引用深度：≤1层
- ✅ 命名规范：kebab-case

**Frontend-Design 成绩**：**A+（100%）** ✅

**A**: 采用 PDA 三层架构：
1. 将详细内容移至 references/ 目录
2. SKILL.md 仅保留核心描述和引用链接
3. 使用紧凑表格提升信息密度
4. 多轮精简优化

#### Q2: 如何确保版本号一致？

**A**: 遵循以下策略：
1. 使用语义化版本规范

3. 代码变更时同步更新文档
4. 定期检查版本号一致性

3. P0和P1立即修复
