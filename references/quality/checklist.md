# 质量检查清单

> ✅ **完整检查清单** - 确保代码质量和用户体验

---

## 📖 核心概念

本检查清单涵盖前端开发的所有质量维度，确保代码、设计、性能、无障碍性和安全性符合高标准。

**质量目标**：
- 代码覆盖率 > 80%
- WCAG AA 无障碍标准
- LCP < 2.5s，FID < 100ms，CLS < 0.1
- 无安全漏洞
- SEO优化

---

## 🎨 设计检查

### 视觉一致性
- [ ] 色彩使用Design Token定义
- [ ] 字体使用Design Token定义
- [ ] 间距使用Design Token定义
- [ ] 所有组件风格统一

### 响应式设计
- [ ] 移动优先设计
- [ ] 断点合理设置（320px、768px、1024px、1440px）
- [ ] 横屏和竖屏适配
- [ ] 触摸目标 ≥ 44×44px
- [ ] 图片自适应（srcset或picture元素）

### 布局完整性
- [ ] 所有元素在视口内可见
- [ ] 无水平滚动条（除非必要）
- [ ] 网格系统一致
- [ ] 空白使用合理

### 交互状态
- [ ] 所有交互元素有hover状态
- [ ] 所有交互元素有active状态
- [ ] 所有交互元素有focus状态
- [ ] 禁用状态视觉明显

- [ ] TypeScript严格模式启用

- [ ] 边界条件处理

- [ ] 用户友好的错误消息

### 性能优化

- [ ] 图片优化（WebP、压缩）

### 状态管理

- [ ] 本地状态和全局状态分离

## ♿ 无障碍检查

### 键盘导航
- [ ] 所有交互元素可键盘访问
- [ ] Tab键顺序符合逻辑
- [ ] 焦点指示器清晰可见
- [ ] Esc键关闭模态框和下拉菜单
- [ ] 焦点陷阱（模态框）

### 屏幕阅读器
- [ ] 语义化HTML元素
- [ ] `aria-label`或`aria-labelledby`正确使用
- [ ] `aria-describedby`关联错误消息
- [ ] `role`属性正确使用
- [ ] 状态变化有`aria-live`区域

### 颜色对比度
- [ ] 正文文本对比度 ≥ 4.5:1
- [ ] 大文本对比度 ≥ 3:1
- [ ] UI组件对比度 ≥ 3:1
- [ ] 焦点指示器对比度 ≥ 3:1

### 表单无障碍
- [ ] 所有输入框有关联的label
- [ ] 必填字段有`required`或`aria-required`
- [ ] 错误消息与控件关联（`aria-describedby`）
- [ ] 验证错误有`aria-invalid="true"`
- [ ] 成功消息有`role="status"`

### 图像和媒体
- [ ] 所有图像有`alt`文本
- [ ] 装饰性图像有`alt=""`
- [ ] 图像文字有替代文本
- [ ] 视频有字幕
- [ ] 音频有文字稿

### ARIA属性
- [ ] ARIA使用不滥用
- [ ] 原生HTML优先于ARIA
- [ ] `aria-hidden`正确使用
- [ ] `aria-expanded`正确使用（折叠/展开）
- [ ] `aria-current`正确使用（导航）

---

## ⚡ 性能检查

### Core Web Vitals
- [ ] LCP（最大内容绘制） < 2.5s
- [ ] FID（首次输入延迟） < 100ms
- [ ] CLS（累积布局偏移） < 0.1

### 加载性能
- [ ] 首屏内容快速渲染
- [ ] 关键CSS内联
- [ ] 非关键JS延迟加载
- [ ] 预加载关键资源（`<link rel="preload">`）
- [ ] 预连接到第三方域名（`<link rel="preconnect">`）

### 运行时性能
- [ ] 帧率 ≥ 60fps
- [ ] 无长任务（> 50ms）
- [ ] 内存使用合理
- [ ] 无内存泄漏

### 资源优化
- [ ] 图片压缩和格式优化（WebP）
- [ ] 字体子集化
- [ ] CSS和JS压缩
- [ ] Gzip/Brotli压缩启用
- [ ] CDN使用

### 网络性能
- [ ] HTTP/2或HTTP/3启用
- [ ] 资源缓存策略
- [ ] 减少HTTP请求
- [ ] Service Worker缓存
